A Brief History of Open Source

1967 - IBM Airline Control Program

  • Academic influence
  • Hardware and software tightly coupled
  • Open Source by necessity
  • "Software Business" doesn't exist yet

1976 - "Open Letter to Hobbyists"

  • Hardware beginning to standardize
  • Software becoming expensive to produce
  • Software starting to be sold

1985 - Free Software Foundation

  • Created to protect user freedom
  • Retain academic spirit of openness and sharing
  • "Social movement"; not focused on business

1998 - Netscape Open Sources their Browser

  • Software business is king
  • Rise of the web
  • "Open Source" is coined; more business friendly

2014 - Microsoft Open Sources .NET Core

  • Opposite of "Open Letter to Hobbyists"
  • Large corporate backing and support
  • Business friendly licenses prevail
  • Alternate business models

Future of Open Source?

  • More business involvement
  • More alternative business models
  • "Business friendly" licenses seem to be a winner

Understand Licenses (2 types)

Copyleft

  • MUST SHARE changes publicly
  • GNU General Public License (GPL) is most common

Copyfree / Permissive software license

  • NOT REQUIRED to share changes
  • MIT, BSD, ISC, Apache, Eclipse, and many others

Lessons Learned

README Importance

  • Communicate intent
  • What is the problem?
  • How are you solving it?
  • Any pitfalls?

Sort files alphabetically on a USB drive

My car stereo supports playing mp3 files off of a USB thumbdrive. Unfortunately, it doesn't always sort files and folders alphabetically.

The purpose of this project is to find a fix for this problem.


Python implementation of Parinfer

Parinfer is a novel way of editing LISP code that infers code structure from indentation.

Python is a popular programming language used by many text editors for plugins and extensions.

Having a Parinfer implementation written as a Python library opens the door for more text editors to use Parinfer.
