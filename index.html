What is Open Source?

You are legally allowed to view, modify, and run software code.

In most cases, you are additionally allowed to share your modifications publicly.

A Brief History of Open Source

1967 - IBM Airline Control Program

  • Academic influence of openness and collaboration
  • Hardware and software tightly coupled
  • Open Source by necessity
  • "Software Business" doesn't really exist yet

1976 - "Open Letter to Hobbyists"

  • Hardware becoming more standardized
  • "Software Business" is starting to become A Thing
  • Computers beginning to spread outside of academia and large corporations

1985 - Free Software Foundation

  • Created to protect user freedom
  • Retain academic spirit of openness and sharing
  • "Social movement"; not focused on business

1998 - Netscape Open Sources their Browser

  • Software is king, and Microsoft dominates the industry
  • Rise of the web, where Open Source has always been popular
  • The term "Open Source" is coined with the goal of being business friendly

2014 - Microsoft Open Sources .NET Core

  • Almost the total opposite of "Open Letter to Hobbyists"
  • Open Source enjoys large corporate backing and support
  • Business friendly licenses prevail
  • Businesses creating value using software, but not selling software directly

What is the Future of Open Source?

  • Hopefully more businesses continue to recognize the benefit of Open Source
  • Business models detached from selling software directly
  • "Business friendly" licenses seem to be a winner
  • No one can know what the future holds

Advice & Lessons Learned

Grain of Salt

Understand Licenses (2 types)

Copyleft

  • GNU General Public License (GPL) is most common
  • You are free to use and modify the software, but must also share any changes publicly.

Copyfree / Permissive software license

  • MIT, BSD, ISC, Apache, Eclipse, and many others
  • You are free to use and modify the software, and are not required to share any changes you have made.

Project Scope and Rationale

  • What problem are you trying to solve?
  • Why are you attempting to solve the problem in this way?
  • Hard to predict how people will use your project. Be thorough.
  • Best practices will be different for different project scopes.
  • Define scope in "Issue #1" and README

Issue #1 - Sort files alphabetically on a USB drive

My car stereo supports playing mp3 files off of a USB thumbdrive. Unfortunately, it doesn't always sort files and folders alphabetically.

The purpose of this project is to find a fix for this problem.

Issue #1 - Python implementation of Parinfer

Parinfer is a novel way of editing LISP code that infers code structure from indentation.

Python is a popular programming language used by many text editors for plugins and extensions.

Having a Parinfer implementation written as a Python library opens the door for more text editors to use Parinfer.

Project Size and Implications

  • Pervasive problem in our industry

  • Make your team better
  • Learn to negotiate
  • Best practices will be different for different project sizes.

On Communication

  • Embrace asynchronous communication
  • Escalate medium as needed
  • Assume a multiplier (1:100 ratio)
  • Be explicit and nice
  • Assume good intentions

Play Nice in the Sandbox

  • Pervasive problem in our industry
  • Has a direct impact on the software you write
  • Make your team better
  • Learn to negotiate
  • Remember that your goal is for people to use your work, not learn your weird conventions

Contribute Any Way You Want

  • You do not have to even contribute code
